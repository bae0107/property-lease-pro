### 关于IAM

- 自己实现基于数据库的用户名和密码
- 需要统一认证是合法用户 gateway
- 需要统一鉴权功能权限
    - aop统一实现，模块化为sdk，各个服务包括这个sdk 统一能力

对于main-service，不同的module需要对外提供不同的API，有进程内的internal API，有其他服务需要的远程 API

> 需要考虑把这2种方式的对外API统一化 屏蔽进程内部调用还是远程调用
> - 提供统一的client
    >
- 远程：基于yaml编译出来接口 装入使用
>   - 本地：直接装入接口使用

| 场景                      | 使用 token            | IAM 校验                           | 安全机制           |
|-------------------------|---------------------|----------------------------------|----------------|
| 用户直接访问                  | user-token          | 用户身份                             | JWT + 公钥/私钥签名  |
| 服务A → 服务B internal/user | internal-user-token | 上下文 UserPrincipal + 服务身份         | JWT 对称签名/非对称签名 |
| 系统触发 → SYSTEM token     | system-token        | 只能由 system-service 签发，IAM 校验服务身份 | 非对称签名 / mTLS   |
